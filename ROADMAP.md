# roadmap

timeline of where solstice is going and when.

## when we're doing what

- **started thinking about it:** april 23, 2026
- **actual development starts:** june 2026
- **alpha target:** september 2026
- **1.0 target:** january 2027

yeah, it's aggressive, but doable if i stay focused.

## phase 1: bootstrap (month 1-2, june-july)

get the fundamentals working. can you actually compile stuff?

- [ ] package manager actually works
- [ ] core packages compile without dying
- [ ] system boots to shell (not graphical, just shell)
- **goal:** you can compile firefox from source

this is the make-or-break phase. if the bootstrap doesn't work, nothing else matters.

## phase 2: expand (month 2-4, july-september)

now we add more recipes and start getting a real desktop environment.

- [ ] 50-100 recipes working
- [ ] x11 and basic desktop utilities
- [ ] documentation that's actually useful
- **goal:** xfce or openbox boots and is usable

at this point you can actually use the distro, even if it's basic.

## phase 3: desktop (month 4-6, september-november)

flesh out the packages. make it a legit distro.

- [ ] 150+ recipes
- [ ] common tools (browsers, editors, utilities)
- [ ] install process is documented
- **goal:** a real, usable linux distro with a desktop

this is where it stops feeling like a hobby project and starts feeling like an actual os.

## phase 4: community (month 6+, december onwards)

let the community in. this is where it gets interesting.

- [ ] overlay system is live and working
- [ ] first community overlays start appearing
- [ ] annual awards for best overlays
- **goal:** someone else makes an overlay and it wins an award

when this happens, i know it's actually working.

## year 2 and beyond

- [ ] 500+ community recipes across overlays
- [ ] multiple official overlays that are really polished
- [ ] stable release cycle (we know when updates drop)
- [ ] real community growing around it

if we get here, it's real.

## why this timeline?

june start because that's when i can dedicate real time to this. september alpha because 3 months of bootstrap + expansion should be enough. january 1.0 because 7 months is reasonable for a usable distro. and then community stuff takes over.

is it tight? yeah. but i've got the hardware and the time starting june.
